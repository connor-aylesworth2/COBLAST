Data location

COBLAST+ stores everything it builds — BLAST databases, SRA downloads, search results, the database registry, and temporary scratch files — under a single data folder. Point it at a drive with plenty of free space to keep large databases off your system drive.

Active location: {{ data_dir }} (free space on this drive: {{ free_label }})

{% if pending_dir %}

A new location is saved and will be used after you restart: {{ pending_dir }}

{% endif %}
  • Takes effect on restart. Close COBLAST+ and start it again after saving.
  • No spaces in the path. BLAST+ cannot build databases under a folder whose path contains spaces (for example D:\My Data). Choose a space-free folder such as D:\COBLAST_data.
  • Must be an NTFS drive. BLAST+ cannot build databases on FAT/exFAT drives (typical for USB sticks and SD cards). When you save, COBLAST+ builds a tiny test database here to confirm the drive works.
  • Existing data is not moved. Changing the location starts a fresh database registry at the new folder. Databases built at the old location stay on disk and keep working — re-add them via Databases → Register existing.

SRA reads location (pull-from)

Optionally keep downloaded SRA reads (.sra/.fasta) on a different drive from the databases — for example a roomy USB drive. Only the reads live here; BLAST databases are always built on the data drive above, so this drive does not need to be NTFS and an exFAT/FAT USB stick works.

Active reads location: {{ sra_reads_active }} (free space on this drive: {{ sra_reads_free_label }})

  • Applies immediately. No restart needed — the next fetch and the SRA Workbench scan pick it up right away.
  • Blank means same drive. With no reads folder set, reads and databases share the data drive (the default).
  • Two cards per run when split. Reads appear under this drive and the built database under the data drive in the SRA Workbench; register the database from its card.
